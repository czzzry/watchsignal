Scoring V2: 14/14 issues done

WatchSignal Scoring V2 Architecture

V2 is a whole recommender upgrade. It enriches profiles, enriches candidates, separates retrieval from ranking, handles negative evidence, reconciles shared sessions, and explains confidence. Nudges are only one input.

Inputs

User Profile Evidence

Onboarding seeds, Taste Lab ratings, watched history, shortlist reactions, post-watch outcomes, and freshness.

↓

Session Intent

Audience mode, session mode, active nudge, person requests, runtime needs, language posture, and rewatch rules.

Candidate Supply

Watchability Gate

Prime Germany access, provider access type, media type, watched-state rules, and safe-pick classification stay upstream.

↓

Candidate Retrieval

TMDb discover, person search, keyword retrieval, and later optional retrieval adapters broaden the pool without choosing winners.

Enrichment And Scoring

Concept Registry

Genres, keywords, overview text, runtime, cast, MovieLens tags, and nudge phrases map into stable concepts.

↓

V2 Scoring Core

Per-user affinity, negative preferences, source reliability, recency, confidence, fallback state, and evidence families.

Outputs

Shared Reconciliation

Husband-first, wife-first, compromise, veto risk, overlap strength, bridge value, and repeated mismatch suppression.

↓

Ranked Shortlist

Five picks when enough titles exist, plus dominant reasons, dominant penalties, confidence, partial support, and debug history.

Not Just Nudges Natural-language nudges compile into structured signals, but profile memory and candidate enrichment are equally central.
Profile Enrichment The profile model should learn concepts from Taste Lab, history, post-watch feedback, and repeated patterns, not just genres.
Replaceable And Honest V1 stays available, V2 remains structured and testable, and weak matches produce confidence or fallback explanations.
